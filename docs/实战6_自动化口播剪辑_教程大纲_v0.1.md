## 第 0 节 · 成片先行 + 你需要准备什么

- 开场直接放成片：30 秒预览版（已有 `Codex保姆级教学_30s预览版_1080p.mp4`），再给全片链接
- 一张「原片 vs 成片」对比图：左边是素人口播原片，右边是带卡片/图表/进度条/字幕的成片同一帧
- 素材清单（学员自查表，共 4 样）：
  1. 口播原片 1 支（手机拍摄即可，本案例 3 分 50 秒）
  2. 口播文稿（讲了什么，一份 txt/md）
  3. 演示素材（录屏或截图，可选）
  4. BGM 1 首（可选；可衔接母教程 Suno/CLI 章节生成）
- 环境要求一句话带过：Codex 会自己安装 ffmpeg / Python 依赖，不需要学员手动配环境

**需要素材**：30s 预览成片（已有）、对比帧截图 1 张

## 第 1 节 · 核心理念：蒙眼剪辑法（1 分钟讲明白）

- 为什么不用剪映/PR：不是软件不好，是「学软件」这一步把 90% 的人挡在门外
- 角色分工比喻（对齐母教程「你是导演」的叙事）：
  - 你 = 导演，只做两件事——**看画面、提意见**
  - Codex = 剪辑师，把你的意见翻译成参数和代码
- 关键机制一句话：**截图反馈就是可执行规格**——你不满意的地方，截图 + 圈出来 + 说清楚想要什么，Codex 就能精确修改（复盘二次验证过的方法论）
- 流程总览图（一张流程卡，可复用成片里的流程图风格）：
  `字幕识别 → 时间轴方案 → 代码渲染审核版 → 截图反馈迭代 → 质检 → 1080p 交付`

**需要素材**：流程总览图 1 张（可从成片 P6/P7 图表改）

## 第 2 节 · 第一步：字幕识别与校对

- 把原片扔给 Codex，提示词模板：
  > 「这是我的口播原片和文稿，帮我做词级字幕识别，用文稿校对识别错误，输出 SRT」
- 讲清学员要做的唯一人工活：**通读一遍字幕，改错别字**（AI 识别 95% 准，剩下 5% 只有你能判断）
- 本案例实证：faster-whisper large-v3 词级识别 + 人工校对一轮
- 小贴士：字幕就是后面所有卡点的「时间坐标系」，这一步偷懒，后面全歪

**需要素材**：字幕工作区截图（识别结果 + 校对前后对比）

## 第 3 节 · 第二步：让 Codex 出剪辑方案（你只管审）

- 提示词模板：
  > 「基于字幕时间轴，帮我出一份剪辑方案：哪句话配什么卡片/图表/演示画面、BGM 从哪进、章节怎么分，输出成 md 文档给我审」
- 强调审核点 3 个（学员照抄即可）：
  1. 每个画面提示是不是真的在讲这句话（素材真值）
  2. 演示画面出现的时机对不对
  3. 章节划分是否均匀（本案例踩坑：语义分段长短不一 → 进度条忽快忽慢，后来改为近等长九章节）
- 本案例实证：剪辑方案 v0.2 一轮审核通过后才开工

**需要素材**：剪辑方案 md 截图 1 张

## 第 4 节 · 第三步：代码渲染第一版（config 驱动）

- 提示词模板：
  > 「按这份方案实现渲染产线：所有位置/尺寸/时间/文案写进一个 config 文件，用 Python 生成卡片图表素材，ffmpeg 合成，先渲染 540p 审核版给我看」
- 给学员解释 config 是什么（一屏讲完）：一个「剪辑参数表」，以后所有修改都改这张表，不重做工程
- 展示实际产物结构：`config.py` + `render_codex_tutorial.py` + 生成的卡片/图表 PNG
- 关键习惯：**先渲染 540p 小样审核，通过了才渲 1080p**（省时间省额度）

**需要素材**：config.py 片段截图、540p 审核版首帧截图

## 第 5 节 · 第四步：截图反馈迭代（本章重头戏）

- 这是学员最需要学会的一步，用本案例的真实迭代当教材：
  - 反馈公式：**哪一秒 + 哪一层 + 想要什么样**（复盘操作规则原文）
  - 真实案例走廊（每个配「反馈截图 → 修复后截图」对比，从 git 历史选 4 个）：
    1. 「字幕贴底边了」→ 调 MarginV 垂直居中（v2.1）
    2. 「画面太素」→ 卡片弹入动效 + 26 张信息卡 + 4 组递进图表（v2.2）
    3. 「流程图挡住人了」→ 缩成 640×300 左右交替（v2.4）
    4. 「录屏和讲的内容对不上」→ 换成真实结果截图 + 顺序高亮（v2.5）
- 迭代纪律 2 条：一次只修一个问题；修完抽同一时间点复核
- 顺带交代「安全区」概念（一张示意图）：人物、字幕、进度条三块禁区先锁死，卡片图表只在左右侧栏轮换

**需要素材**：4 组前后对比截图（verify/ 目录已有大部分）、安全区示意图 1 张

## 第 6 节 · 第五步：交付质检 + 归档

- 一句话立规矩：**「渲染成功」不等于「成片正确」**——ffmpeg 跑完返回 0 也可能有错
- 提示词模板（一键质检）：
  > 「渲染 1080p 最终版，然后做完整质检：ffprobe 查时长分辨率、全片解码查损坏、检查响度、抽关键帧截图给我确认，归档时校验哈希」
- 学员只需看 Codex 返回的质检报告 + 抽帧截图，点头即交付
- 加餐：让 Codex 顺手切一段 30 秒连续预览（选完整语义段，不要拼贴）用于发布引流

**需要素材**：质检报告输出截图、响度数据截图

## 第 7 节 · 避坑清单 + 沉淀成 Skill

- 避坑清单（5 条，来自复盘「没有一次做对的」，每条一行）：
  1. 字幕位置第一版必歪，直接要求「垂直居中于底部安全区」
  2. 章节要近等长，否则进度条变速很难受
  3. 大图表别放画面中心，会挡人
  4. 录屏对不上口播就换截图，别硬移高亮框
  5. 小字看不清和文字被裁切是两个问题，分开报
- 呼应母教程实战7 的收尾动作——沉淀 Skill：
  > 「帮我把这次口播剪辑的完整流程沉淀成一个可复用的 skill，命名为 talking-head-cut」
- 结尾承接：下次换一支口播，直接调 skill，一句话开剪

**需要素材**：skill 沉淀对话截图 1 张
