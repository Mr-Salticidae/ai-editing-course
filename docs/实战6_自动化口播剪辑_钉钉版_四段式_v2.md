## 二、所用工具

### 1. Codex——总执行者

OpenAI 的 AI 智能体。它不只是回答问题，而是能在你电脑上直接干活：读你的素材、写剪辑代码、调用工具、渲染成片、自己做质检。本教程里所有的剪辑动作都是它执行的，你只负责看和拍板。

官网地址：chatgpt.com/codex（需科学上网，用 ChatGPT 账号登录）

安装方法见实战2，装过就不用再装。

适用场景：口播视频剪辑、批量图文处理、写代码、做表格 PPT 等需要"动手"的任务。

### 2. 四个 skill——Codex 的剪辑班底

剪辑不是 Codex 一个人干的，它背后是一支各管一摊的 **AI 乐队**：

![图3 4个skill班底总览](https://raw.githubusercontent.com/Mr-Salticidae/ai-editing-course/main/docs/images/%E5%AE%9E%E6%88%986/%E5%9B%BE03_skill%E7%8F%AD%E5%BA%95%E6%80%BB%E8%A7%88.png)

**blind-editing-workflow（蒙眼剪辑法）——剪辑师本体。** 整条产线的执行核心：给你出剪辑方案、生成画面元素、合成成片，并且按你的截图反馈改片。这一章你打交道最多的就是它。

**song-caption-mv-workflow（歌曲字幕）——字幕组。** 负责把你说的话变成带时间轴的字幕。你不用单独召唤它，剪辑实操第一步会自动用到。

**suno-music-brief（Suno 配乐）——音乐总监，选配。** 想给片子配 BGM 时，让它帮你写 Suno 提示词；不配乐就完全用不到它。

**remotion-skill（Remotion 知识卡片）——动效设计师，延伸。** 做图文卡片类视频的姊妹技能。本期口播片它没上场，以后想做纯图文视频（没有真人出镜那种）再召唤。

安装方式你在实战 2 已经会了：把素材包发给 Codex，说一句 **帮我安装这几个 skill** 即可。

![图4 安装skill对话](https://raw.githubusercontent.com/Mr-Salticidae/ai-editing-course/main/docs/images/%E5%AE%9E%E6%88%986/%E5%9B%BE04_%E5%AE%89%E8%A3%85skill%E5%AF%B9%E8%AF%9D.png)

### 3. 幕后引擎——生成图片和视频的工具

成片里的字幕、卡片、图表、进度条不是凭空来的，是下面这三样做出来的。**它们都由 Codex 自动调用，你不用自己安装、也不用学**，列在这里只是让你知道每样东西是谁做的：

**faster-whisper（large-v3 模型）——出字幕。** 把你的口播语音识别成带词级时间戳的字幕。本片所有画面的出现时机都对着这条时间轴，所以它是整支片子的地基。项目地址：github.com/SYSTRAN/faster-whisper

**Pillow（Python 图像库）——出画面元素。** 本片的 26 张信息卡片、4 组递进图表、章节进度条和高亮框，全部是它按配置批量生成的透明 PNG 图片。官网地址：python-pillow.org

**FFmpeg——出视频。** 把口播原片、卡片、图表、字幕、BGM 合成为成片，并完成 1080p 编码、抽帧检查和响度质检。官网地址：ffmpeg.org

### 4. Suno——配乐（选配）

AI 音乐生成工具。想给片子加 BGM 时用，提示词交给 suno-music-brief 帮你写；不配乐可以完全跳过。

官网地址：suno.com

## 三、操作方法

主线一共五步。每一步都是「**你说一段话，Codex 给一个结果**」——你只负责看和拍板。

### 1. 把素材丢给 Codex，一句话开剪

把原片、口播稿放进项目文件夹，对 Codex 说：

```
这是我的口播原片、口播稿和对标视频，用蒙眼剪辑法帮我剪成教学片。先做字幕识别，用口播稿校对后给我看。
```

Codex 会自动识别你的语音、再拿口播稿校对一遍，然后把字幕文件给你看。你唯一的活：**通读一遍字幕，改掉错别字。** 别偷懒——这份字幕是后面所有画面卡点的时间轴，它错了后面全歪。

![图5 Step1开剪对话](https://raw.githubusercontent.com/Mr-Salticidae/ai-editing-course/main/docs/images/%E5%AE%9E%E6%88%986/%E5%9B%BE05_%E5%BC%80%E5%89%AA%E5%AF%B9%E8%AF%9D.png)

![图6 校对好的字幕文件](https://raw.githubusercontent.com/Mr-Salticidae/ai-editing-course/main/docs/images/%E5%AE%9E%E6%88%986/%E5%9B%BE06_%E5%AD%97%E5%B9%95%E6%A0%A1%E5%AF%B9%E6%96%87%E4%BB%B6.png)

### 2. 审剪辑方案

字幕定了，Codex 会回一份剪辑方案：全片分几章、哪句话配什么卡片、图表、演示画面。你只盯两个审核点：

1. **每个画面是不是真的在讲这句话**——画面和口播对不上的，直接指出来；
2. **章节长短是否大致均匀**——某一章特别长或特别短，让它重新分。

没问题就回一句「确认，开始渲染」。

![图7 剪辑方案文档](https://raw.githubusercontent.com/Mr-Salticidae/ai-editing-course/main/docs/images/%E5%AE%9E%E6%88%986/%E5%9B%BE07_%E5%89%AA%E8%BE%91%E6%96%B9%E6%A1%88%E6%96%87%E6%A1%A3.png)

![图8 确认开始渲染对话](https://raw.githubusercontent.com/Mr-Salticidae/ai-editing-course/main/docs/images/%E5%AE%9E%E6%88%986/%E5%9B%BE08_%E7%A1%AE%E8%AE%A4%E6%B8%B2%E6%9F%93%E5%AF%B9%E8%AF%9D.png)

### 3. 看小样

Codex 不会直接渲高清，而是先出一版 540p 的小样给你审——小样审过再渲高清，省时间也省额度。

![图9 540p小样画面](https://raw.githubusercontent.com/Mr-Salticidae/ai-editing-course/main/docs/images/%E5%AE%9E%E6%88%986/%E5%9B%BE09_540p%E5%B0%8F%E6%A0%B7%E7%94%BB%E9%9D%A2.png)

### 4. 截图反馈：导演的核心动作

小样肯定有不满意的地方，这很正常。改片靠一个反馈公式：

**截图 + 圈出来 + 说想要什么**（哪一秒、哪个元素、改成什么样）

两组本片的真实案例：

**案例一：字幕贴到底边了。** 截一张图圈住字幕，说「字幕太靠下了，在底栏里居中」——下一版字幕就位。

![图10 案例一字幕位置前后对比](https://raw.githubusercontent.com/Mr-Salticidae/ai-editing-course/main/docs/images/%E5%AE%9E%E6%88%986/%E5%9B%BE10_%E6%A1%88%E4%BE%8B%E4%B8%80%E5%AD%97%E5%B9%95%E4%BD%8D%E7%BD%AE%E5%89%8D%E5%90%8E%E5%AF%B9%E6%AF%94.png)

**案例二：流程图挡住人了。** 大图表压在画面中间，把讲师整个盖住。截图圈出来，说「图表缩小，移到两侧，别挡人」——下一版图表变小、左右交替出现。

![图11 案例二图表挡人前后对比](https://raw.githubusercontent.com/Mr-Salticidae/ai-editing-course/main/docs/images/%E5%AE%9E%E6%88%986/%E5%9B%BE11_%E6%A1%88%E4%BE%8B%E4%BA%8C%E5%9B%BE%E8%A1%A8%E6%8C%A1%E4%BA%BA%E5%89%8D%E5%90%8E%E5%AF%B9%E6%AF%94.png)

一条迭代纪律：**一次只提一个问题，改完确认了再提下一个。** 一口气提五个，改乱了你都不知道是哪条改坏的。

### 5. 成片交付

小样满意后，最后一句话：

```
渲染 1080p 最终版，做完质检后把报告和关键帧截图发我确认。
```

Codex 会渲染成片、自己做一遍质检，把结论和几张抽帧图发给你。你看抽帧没问题，点头，交付完成。顺手再让它切一段 30 秒预览，发布时当预告用——本教程开头那 30 秒就是这么来的。

## 四、总结

这套流程适合**有完整口播原片、需要加字幕和信息图的教学片、口播片、课程片**。它的优势不是"AI 替你审美"，而是**你不用碰剪辑软件，也能把每一处不满意精确改掉**：所有画面元素都是代码生成的，位置、大小、时间都能一句话调整，改一百遍也不会乱。

**这套班底可以复用。** skill 装一次就够了。下支口播视频，依然是：丢素材 → 一句话开剪 → 截图当导演。流程跑熟之后，还可以像实战 7 教的那样，把你自己的习惯沉淀成专属 skill——你的班底会越用越懂你。

### 小技巧

**字幕一定要自己通读一遍。** 它是全片的时间轴地基，错一个字，后面所有画面卡点都跟着歪。这是全流程里唯一必须你亲手做的活。

**一次只提一个问题。** 改完确认了再提下一个，否则改乱了都不知道是哪条改坏的。

**先看 540p 小样，别急着渲高清。** 小样审过再出成片，省时间也省额度。

**反馈要带三要素：哪一秒、哪个元素、改成什么样。** 只说"不好看"，AI 只能瞎猜；截图圈出来说"图表缩小移到两侧"，一次就能改对。

**画面别压住中心人物。** 口播片先给人物、字幕、进度条留出安全区，卡片和图表只在左右两侧轮换出现。

**讲界面结构时，清晰的截图往往比录屏更好用。** 录屏里画面跳变、内容和旁白对不上时，换成准确的结果截图加高亮框，观感更真实。讲操作过程才必须用录屏。

**成片别只看"能播放"。** 让 Codex 出质检报告：时长、分辨率、响度、关键帧截图都对上，才算交付完成。

**顺手切一段 30 秒预览。** 发布时当预告用，不用另外剪。
